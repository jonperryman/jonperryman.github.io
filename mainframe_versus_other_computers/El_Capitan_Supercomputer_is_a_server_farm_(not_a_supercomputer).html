I asked Gemini "how many blade servers in El Capitan supercomputer"

El Capitan contains approximately 5,568 compute blades.

Here's how we arrive at that number:

  • El Capitan has 11,136 compute nodes.

  • Each compute blade contains 2 compute nodes.

  • Therefore, the total number of blades is 11,136 nodes/2 nodes/blade=5,568 blades.

These blades are part of the HPE Cray EX system and are densely packed to achieve the supercomputer's immense processing power.1 Each blade houses multiple AMD Instinct MI300A APUs, which are the core computational units.2

I asked Gemini "does el capitan use linux"

Yes, El Capitan absolutely uses Linux as its operating system.1

More specifically, it uses TOSS (Tri-Lab Operating System Software), which is a custom Red Hat Enterprise Linux (RHEL) distribution developed by Lawrence Livermore National Laboratory (LLNL) and the other National Nuclear Security Administration (NNSA) labs (Los Alamos and Sandia).2

The choice of Red Hat Enterprise Linux as the base for TOSS provides a robust, stable, and widely supported foundation, while TOSS itself allows for specialized customizations and optimizations necessary for managing an exascale supercomputer and its unique hardware (like the AMD MI300A APUs and HPE Slingshot interconnect). This standardization also helps streamline administration and provides a consistent user experience across different systems within the NNSA complex.

I asked Gemini "how much memory does el capitan have"

El Capitan has a substantial amount of memory, designed to handle its exascale computing tasks.1 The total memory in El Capitan is 5.4375 petabytes (PB).2 💾

This massive memory capacity is distributed across its architecture as follows:

Memory Per Node

Each compute node in El Capitan features 512 GB of HBM3 memory. This is a significant amount of high-bandwidth memory.3

The Role of AMD Instinct MI300A APUs

The memory architecture is built around the AMD Instinct MI300A Accelerated Processing Units (APUs).4

  • Each MI300A APU integrates CPU cores, GPU compute units, and 128 GB of unified HBM3 memory onto a single package.5

  • Since each compute node in El Capitan contains four of these MI300A APUs, the total memory per node is 4×128 GB=512 GB.

This unified memory design allows the CPU and GPU components within each APU to share the same high-bandwidth memory, which significantly reduces data transfer bottlenecks and improves efficiency for complex, data-intensive workloads common in HPC and AI.6

Total System Memory

With 11,136 compute nodes in the system, the total memory capacity is:

11,136 nodes×512 GB/node=5,701,632 GB

Converting this to petabytes:

5,701,632 GB÷1024 GB/TB÷1024 TB/PB≈5.4375 PB

This vast amount of high-bandwidth memory is crucial for enabling El Capitan to achieve its exascale performance and tackle some of the world's most challenging scientific and national security problems.7

I asked Gemini "where are the disks in el capitan"

El Capitan employs a multi-tiered storage system to handle the massive amounts of data generated by its exascale computations.1 The "disks" in El Capitan are primarily located in two key areas:

1. Near-Node Local Storage (The "Rabbits") 🐇2

El Capitan utilizes innovative near-node local storage modules aptly nicknamed "Rabbits." These modules are installed alongside El Capitan's compute nodes within the same HPE Cray EX racks.3 The purpose of these "Rabbits" is to provide extremely fast, high-bandwidth storage that is physically very close to the compute nodes.4 This significantly reduces data transfer times for applications that need to read and write data frequently and quickly during their execution.

Think of it like having a small, super-fast scratchpad right next to where you're doing intense calculations. Data can be moved to and from this scratchpad almost instantaneously, minimizing delays.

2. Global Lustre File System 🌐

While the "Rabbits" provide rapid, temporary storage, El Capitan also connects to a global Lustre file system.5 This is a larger, shared, and longer-term storage solution that all compute nodes can access. Once an application completes its work on the "Rabbits," the data can be migrated to this global Lustre file system for more permanent storage and access by other applications or for archival purposes.6

This two-tiered approach allows El Capitan to optimize data movement:

  • "Rabbits": For high-speed, in-situ (on-the-fly) data processing and checkpointing during active computations.

  • Global Lustre: For long-term storage, sharing data across the entire system, and managing larger datasets.7

This architecture is crucial for handling the immense data generated by exascale simulations, which can produce petabytes of data from a single run.

I asked Gemini "how many blade servers connected to each rabbit"

In El Capitan, one Rabbit module is connected to eight compute blades, which contain a total of 16 compute nodes.

Here's the breakdown:

  • An El Capitan compute chassis houses 8 compute blades.1

  • Each compute blade contains 2 compute nodes.

  • Therefore, each chassis holds 8 blades×2 nodes/blade=16 compute nodes.

  • One Rabbit module is deployed per compute chassis, directly connected to these 8 compute blades (16 compute nodes) via PCIe.2

This architecture ensures that the high-speed local storage provided by the "Rabbits" is readily available to a dedicated group of compute nodes, minimizing latency and maximizing data throughput for the most demanding workloads.
