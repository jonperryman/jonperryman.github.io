El Capitan employs a multi-tiered storage system to handle the massive amounts of data generated by its exascale computations.1 The "disks" in El Capitan are primarily located in two key areas:




1. Near-Node Local Storage (The "Rabbits") üêá2



El Capitan utilizes innovative near-node local storage modules aptly nicknamed "Rabbits." These modules are installed alongside El Capitan's compute nodes within the same HPE Cray EX racks.3 The purpose of these "Rabbits" is to provide extremely fast, high-bandwidth storage that is physically very close to the compute nodes.4 This significantly reduces data transfer times for applications that need to read and write data frequently and quickly during their execution.


Think of it like having a small, super-fast scratchpad right next to where you're doing intense calculations. Data can be moved to and from this scratchpad almost instantaneously, minimizing delays.



2. Global Lustre File System üåê



While the "Rabbits" provide rapid, temporary storage, El Capitan also connects to a global Lustre file system.5 This is a larger, shared, and longer-term storage solution that all compute nodes can access. Once an application completes its work on the "Rabbits," the data can be migrated to this global Lustre file system for more permanent storage and access by other applications or for archival purposes.6


This two-tiered approach allows El Capitan to optimize data movement:

  • "Rabbits": For high-speed, in-situ (on-the-fly) data processing and checkpointing during active computations.

  • Global Lustre: For long-term storage, sharing data across the entire system, and managing larger datasets.7


This architecture is crucial for handling the immense data generated by exascale simulations, which can produce petabytes of data from a single run.
