CPU caches are an essential part of the memory hierarchy in modern computer systems, designed to bridge the significant speed gap between the very fast CPU and much slower main memory (RAM).1 They store frequently used data and instructions close to the processor to minimize latency.2


Here's a breakdown of the typical cache levels and their location:



Caches Located Inside the CPU (On-Die)



Almost all modern CPUs integrate multiple levels of cache directly onto the processor die.3 These are the fastest and smallest caches.


  1. L1 Cache (Level 1 Cache):

    • Location: This is the smallest, fastest, and most expensive cache. It is always located directly within each CPU core.

    • Characteristics:

      • Extremely fast access, typically 1-3 CPU clock cycles.4

      • Smallest capacity, usually between 32KB and 128KB per core.5

      • Often divided into two parts:

        • L1 Instruction Cache (L1i): Stores frequently accessed instructions.

        • L1 Data Cache (L1d): Stores frequently accessed data.

    • Purpose: To provide the CPU core with immediate access to the data and instructions it needs right now, minimizing stalls.6

  1. L2 Cache (Level 2 Cache):

    • Location: The L2 cache is also almost always located on the CPU die, usually slightly further from the core than L1 but still highly integrated.

    • Characteristics:

      • Larger than L1, typically ranging from 256KB to 2MB per core.

      • Slower than L1 (e.g., 10-20 CPU cycles) but still significantly faster than RAM.7

      • Can be private (one L2 per core) or shared among a few cores, depending on the CPU architecture.8

    • Purpose: To serve as a backup for the L1 cache, storing more data and instructions that might be needed soon but didn't fit in L1, thus reducing trips to the slower L3 or main memory.9

  1. L3 Cache (Level 3 Cache):

    • Location: The L3 cache is also located on the CPU die. It is generally the largest cache level within the CPU.

    • Characteristics:

      • Much larger than L1 and L2, ranging from several MB to hundreds of MB (e.g., 16MB, 32MB, 64MB, or even more in high-end processors like AMD's X3D CPUs with V-Cache).

      • Slower than L1 and L2, but still much faster than main memory (e.g., 30-60 CPU cycles).

      • Almost always shared by all cores on the CPU die. This allows different cores to easily share data without having to go out to main memory.

    • Purpose: To reduce the average memory access time for data that isn't found in L1 or L2, acting as a large pool of frequently accessed information for all cores.10




Caches Located Outside the CPU (Historically, or for very specific purposes)



While the vast majority of CPU cache is now integrated on-die, historically, some levels of cache were located outside the CPU.

  1. Older L2 or L3 Caches (Historical):

    • In the past (e.g., in the Pentium II, early Pentium III, and some older Athlon CPUs), the L2 cache was sometimes located on a separate chip or module on the motherboard, very close to the CPU. This was due to manufacturing limitations at the time.

    • Similarly, very early L3 caches (before they became common on-die) might have been motherboard-based.11

    • Modern relevance: This is largely historical. Almost all L1, L2, and L3 caches in current desktop, laptop, and server CPUs are integrated onto the processor die.

  1. L4 Cache (Rare and Specialized):

    • Location: An L4 cache, when present, is typically located outside the main CPU die but still within the CPU package (e.g., on the same multi-chip module) or sometimes on the motherboard, acting as an intermediary between the CPU's on-die caches and main RAM.12

    • Characteristics:

      • Usually very large (e.g., 128MB or more).

      • Often implemented using eDRAM (embedded DRAM), which is denser than SRAM (used for L1-L3) but slower.

      • Primarily used in specific architectures, for example, some Intel Haswell and Broadwell CPUs with integrated GPUs used eDRAM as an L4 cache (often called "Crystalwell") to provide high-bandwidth memory for both the CPU cores and the powerful integrated graphics. Intel has experimented with L4 again in some newer architectures like Meteor Lake.13

    • Purpose: To serve as a very large victim cache for L3, a shared memory pool for integrated GPUs, or a bridge to main memory to further reduce latency for certain workloads.

    • Modern relevance: L4 cache is not standard on most consumer CPUs. It's a specialized implementation found in particular architectures where a specific performance bottleneck (e.g., integrated graphics memory bandwidth) justifies its inclusion.

In summary:

  • Inside the CPU (on-die): L1, L2, and L3 caches are universally found on the processor chip itself in modern CPUs, ordered by speed (L1 fastest) and size (L3 largest).

  • Outside the CPU: This is largely a historical artifact for L2/L3. L4 cache can be considered "outside" the main CPU die but usually within the same package for specific, niche implementations. Main Memory (RAM) is always physically separate from the CPU, connected via the motherboard's memory channels.
