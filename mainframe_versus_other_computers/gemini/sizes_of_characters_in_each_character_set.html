I asked Gemini: "what are the sizes of characters in each character set?"

The size of characters varies depending on the character set and, importantly, the encoding used for that character set. Here's a breakdown of the sizes in some common character sets:

1. ASCII (American Standard Code for Information Interchange):

  • Size: 7 bits per character.
  • Storage: Typically stored in 1 byte (8 bits). The most significant bit is often 0.
  • Range: 128 characters, including uppercase and lowercase English letters, digits, punctuation marks, and control characters.

2. Extended ASCII:

  • Size: 8 bits per character (1 byte).
  • Storage: 1 byte.
  • Range: 256 characters. Extends the original ASCII to include additional symbols, accented letters, and graphical characters. However, there are various incompatible "extended ASCII" sets (code pages).

3. Unicode:

  • Unicode is a character *set* that aims to assign a unique code point to every character in every known writing system. The size of a character in Unicode itself (the code point) is conceptually 21 bits.
  • However, when Unicode characters are stored or transmitted, they need to be *encoded*. Different Unicode encodings use different numbers of bytes per character:
    • UTF-8 (Unicode Transformation Format - 8-bit):
      • Size: Variable, 1 to 4 bytes per character.
      • Breakdown:
        • 1 byte for ASCII characters (U+0000 to U+007F).
        • 2 bytes for most Latin, Greek, Cyrillic, Hebrew, Arabic, etc. (U+0080 to U+07FF).
        • 3 bytes for most characters in the Basic Multilingual Plane (BMP), including most CJK characters (U+0800 to U+FFFF).
        • 4 bytes for supplementary characters (emoji, less common CJK, etc.) (U+10000 to U+10FFFF).
      • Most widely used encoding for web pages and general text due to its efficiency and backward compatibility with ASCII.
    • UTF-16 (Unicode Transformation Format - 16-bit):
      • Size: Variable, 2 or 4 bytes per character.
      • Breakdown:
        • 2 bytes (16 bits) for most characters in the Basic Multilingual Plane (BMP) (U+0000 to U+FFFF), excluding surrogate code points.
        • 4 bytes (two 16-bit code units called surrogate pairs) for supplementary characters (U+10000 to U+10FFFF).
      • Used internally by some operating systems and programming languages (e.g., Java, Windows internally). Can be either big-endian (UTF-16BE) or little-endian (UTF-16LE).
    • UTF-32 (Unicode Transformation Format - 32-bit):
      • Size: Fixed, 4 bytes (32 bits) per character.
      • Range: Can directly represent all Unicode code points.
      • Advantage: Simplifies character indexing and manipulation due to the fixed width.
      • Disadvantage: Less efficient in terms of storage space, especially for text containing mostly ASCII characters. Can also be big-endian (UTF-32BE) or little-endian (UTF-32LE).

In summary:

  • ASCII: 1 byte per character.
  • Extended ASCII: 1 byte per character (but varies between sets).
  • Unicode: The size depends on the encoding:
    • UTF-8: 1-4 bytes per character.
    • UTF-16: 2 or 4 bytes per character.
    • UTF-32: 4 bytes per character.

When working with text, it's crucial to know both the character set and the encoding being used to correctly interpret the data and determine the storage size of the characters. UTF-8's variable-width nature makes it efficient for diverse text, while UTF-32's fixed width simplifies processing at the cost of space. UTF-16 offers a compromise.
