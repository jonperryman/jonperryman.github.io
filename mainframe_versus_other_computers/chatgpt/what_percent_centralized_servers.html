I asked ChatGPT: "what percentage of servers are centralized?"

There isn't a single hard number for the percentage of servers that are centralized, because it depends heavily on how you define "centralized" — but here’s a clear, honest breakdown:

  • Enterprise servers (companies, governments, universities):
    Over 90% are traditionally centralized — meaning they operate from dedicated data centers, server rooms, or private cloud environments controlled by the organization.
  • Cloud servers (AWS, Azure, Google Cloud, etc.):
    Cloud computing *appears* decentralized to users (because it’s "in the cloud"), but in reality, it's highly centralized inside massive data centers run by a few companies.
    Today, 65–75% of global server workloads are on major cloud platforms, and these are centralized at the infrastructure level.
  • True decentralized networks (blockchains, peer-to-peer systems like BitTorrent):
    These make up a tiny fraction, probably less than 1% of all servers worldwide.
Realistic estimate:

If you define "centralized" as "controlled from a few physical locations or by a small number of organizations," probably 90–95% of servers are centralized today.

Why so centralized?
  • It's cheaper and easier to maintain big server farms.
  • Security and compliance are easier with central control.
  • Performance can be optimized in big data centers.

But: there’s a growing trend toward *decentralized or distributed models* like edge computing, decentralized web (Web3), and blockchain — but these are still small compared to traditional centralized systems.
